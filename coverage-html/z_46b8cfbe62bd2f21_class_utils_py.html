783 @finalmethod 

784 def _is_cas(self, value: Any) -> bool: 

785 """Check if a string is a valid CAS registry number 

786 

787 CAS numbers are always in a format of three segments of numerical values: 

788 1234-56-6 

789 

790 The first segment can be from 2 to 7 intigers (needs to be at least one non-zero value), 

791 and the second is always 2 integers. These are basically just unique numbers, but there's no 

792 established numbering system or other restrictions. 

793 The third segment is one integer, and that is the checksum of the first two segments. 

794 

795 https://regex101.com/r/xPF1Yp/2 

796 (?P<seg_a>[0-9]{2,7})-(?P<seg_b>[0-9]{2})-(?P<checksum>[0-9]) 

797 

798 The checksum is calculated by taking the first two segments and iterating over each 

799 individual intiger in reverse order, multiplying each by its position, then taking 

800 the modulous of the sum of those values. 

801 

802 For example, 1234-56-6 is valid because the result of the below equation matches the checksum, (which is 6) 

803 (6*1 + 5*2 + 4*3 + 3*4 + 2*5 + 1*6) % 10 == 6 

804 

805 This can be simplified in the below aggregation: 

806 cas_chars = [1, 2, 3, 4, 5, 6] 

807 sum([(idx+1)*int(n) for idx, n 

808 in enumerate(cas_chars[::-1])]) % 10 

809 

810 See: 

811 https://www.cas.org/training/documentation/chemical-substances/checkdig 

812 https://www.allcheminfo.com/chemistry/cas-number-lookup.html 

813 

814 Args: 

815 value (str): The value to determine if its a CAS # or not 

816 

817 Returns: 

818 bool: True if its a valid format and the checksum matches 

819 

820 Example: 

821 >>> self._is_cas("1234-56-6") 

822 True 

823 >>> self._is_cas("50-00-0") 

824 True 

825 >>> self._is_cas("1234-56-999") 

826 False 

827 >>> self._is_cas("1234-56") 

828 False 

829 >>> self._is_cas("1234-56-0") 

830 False 

831 >>> self._is_cas("0000-00-0") 

832 False 

833 >>> self._is_cas("00-10-0") 

834 False 

835 """ 

836 

837 if isinstance(value, str) is False: 837 ↛ 838line 837 didn't jump to line 838 because the condition on line 837 was never true

838 return False 

839 

840 # value='1234-56-6' 

841 # https://regex101.com/r/xPF1Yp/2 

842 cas_pattern_check = re.match(r"^(?P<seg_a>[0-9]{2,7})-(?P<seg_b>[0-9]{2})-(?P<checksum>[0-9])$", value) 

843 

844 if cas_pattern_check is None: 

845 return False 

846 

847 cas_dict = cas_pattern_check.groupdict() 

848 # cas_dict = dict(seg_a="1234", seg_b="56", checksum="6") 

849 

850 if int(cas_dict["seg_a"]) == 0: 850 ↛ 851line 850 didn't jump to line 851 because the condition on line 850 was never true

853 cas_chars = list(cas_dict["seg_a"] + cas_dict["seg_b"]) 

854 # cas_chars = ["1","2","3","4","5","6"] 

855 

856 checksum = sum([(idx + 1) * int(n) for idx, n in enumerate(cas_chars[::-1])]) % 10 

857 # checksum = 6

859 return int(checksum) == int(cas_dict["checksum"])
